# Reference

> Methodology, glossary, runbook, sibling-projects, external landscape, and
> the playbook for periodic strategic re-evaluation.

These docs are the "how we work" layer. They don't describe a specific
strategy or platform piece — they describe disciplines, terms, and
processes that apply across the project.

## Documents

| File | What it covers |
|---|---|
| [`stages-and-verdicts.md`](stages-and-verdicts.md) | The strategy pipeline: stage definitions, entry/exit criteria, verdict types, the bar for "non-viable". |
| [`methodology.md`](methodology.md) | Validation rigor: pre-registration, hard date splits, null-shuffle benchmarks, Wilson CIs, walk-forward, kill criteria, full-distribution reporting. |
| [`fresh-eyes-playbook.md`](fresh-eyes-playbook.md) | When to do strategic re-evaluation, how to do it, what the deliverable looks like. The harness for staying out of the fog. |
| [`terminology.md`](terminology.md) | Glossary of trading concepts and project-specific terms. |
| [`runbook.md`](runbook.md) | How to run scripts, install launchd jobs, recover from common failures. |
| [`sibling-projects.md`](sibling-projects.md) | Notes on `kalshi-autoagent`, `kalshi-arb-trader`, `crypto-copy-bot`, `options-autoagent`. What's covered there, what we should not duplicate, lessons we import. |
| [`external-landscape.md`](external-landscape.md) | What's happening in the world: HIP-4 / Kalshi × Hyperliquid partnership, Polymarket, regulatory developments, anything that shifts our assumption set. |
